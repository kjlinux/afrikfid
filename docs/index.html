Node.js 22 Express 4.18 PostgreSQL 16 Redis 7 React 18 CDC v2.0 Compliant Docker Ready

Vue d'ensemble

Afrik'Fid est une plateforme de paiement B2B complète pensée pour les marchés africains. Elle permet aux marchands d'accepter des paiements de leurs clients via Mobile Money ou carte bancaire, tout en offrant automatiquement un programme de cashback (fidélité) aux acheteurs.

Marchands

Intègrent l'API pour accepter des paiements, suivre leurs ventes, gérer les liens de paiement et recevoir leurs virements automatiques.

Clients

Paient via Mobile Money ou carte. Gagnent automatiquement du cashback crédité sur leur portefeuille selon leur niveau de fidélité.

Administrateurs

Supervisent la plateforme : KYC marchands, règles anti-fraude, taux de change, audit, distributions et litiges.

Modèle X/Y/Z — Le coeur d'Afrik'Fid

Toute transaction sur Afrik'Fid suit le modèle X/Y/Z, qui définit comment le montant payé est redistribué entre le marchand, le client et la plateforme.

Formule fondamentale
X% = Remise marchand (pourcentage que le marchand consent à reverser)
Y% = Cashback client (rebate crédité sur le portefeuille du client)
Z% = Commission Afrik'Fid = X% − Y%

Contrainte : Y ≤ X (le client ne peut pas gagner plus que ce que le marchand cède)

Exemple concret :
Achat de 10 000 XOF, marchand configure X=8%, Y=5%
→ Client gagne : 500 XOF de cashback
→ Afrik'Fid perçoit : 300 XOF (Z = 8%-5% = 3%)
→ Marchand reverse : 800 XOF (dont 500 vont au client)

Niveaux de fidélité

Le taux Y% appliqué dépend du niveau de fidélité du client, qui évolue automatiquement selon son historique d'achats.

OPEN
0%
Nouveau client
LIVE
5%
3 achats / 3 mois
GOLD
8%
6 achats / 6 mois
ROYAL
12%
12 achats / 12 mois
i
Les seuils (nombre d'achats, montant cumulé, durée) sont configurables par l'admin depuis le dashboard ou via PUT /api/v1/loyalty/config/:status. Des surcharges par pays sont également possibles.
!
Rétrogradation : Un client inactif pendant la fenêtre de son tier peut redescendre au niveau inférieur. Le batch de fidélité évalue cela chaque nuit.

Tokens JWT

Access Token

Durée : 15 minutes. Envoyé dans le header Authorization: Bearer <token>. Contient le JTI (ID unique) pour révocation.

Refresh Token

Durée : 7 jours. Stocké en Redis. Permet d'obtenir un nouvel access token sans re-login via POST /auth/refresh.

2FA (TOTP)

Les admins et marchands peuvent activer l'authentification à deux facteurs via une application comme Google Authenticator ou Authy.

1. POST /auth/2fa/setup → reçoit secret + QR code (URI otpauth://)
2. L'utilisateur scanne le QR avec son app authenticator
3. POST /auth/2fa/verify avec le code à 6 chiffres → active le 2FA
4. Lors du login suivant, si 2FA actif → doit fournir totp_code dans le body
5. 10 codes de secours single-use générés à l'activation

Protection contre le brute-force

  • 5 tentatives échouées → compte bloqué 15 minutes (via Redis)
  • Rate limiter global : 200 requêtes / 15 min par IP
  • Rate limiter spécifique login : 20 tentatives / 15 min

Sécurité & Chiffrement

Chiffrement AES-256-GCM

Les données sensibles (PII) sont chiffrées en base de données avec AES-256-GCM. Même un accès direct à la DB ne révèle pas ces données.

ChampTableType
Numéro de téléphoneclientsChiffré AES + hash HMAC pour recherche
EmailclientsChiffré AES + hash HMAC pour recherche
Numéro de compte bancairemerchantsChiffré AES + hash HMAC
i
La recherche par téléphone utilise le champ phone_hash (HMAC-SHA256 du numéro) pour éviter de déchiffrer toutes les entrées lors d'une recherche.

Rotation des clés

Chaque jour à 3h00, un worker vérifie si la clé AES a plus de 90 jours. Si oui, elle est automatiquement tournée (conformité PCI-DSS). Les clés API marchands sont également rotées tous les 90 jours (3h30).

Mode Sandbox

Lorsque SANDBOX_MODE=true (ou identifiants opérateur absents), tous les paiements passent par des simulateurs internes. Idéal pour les tests sans risque de vrai mouvement d'argent.

OK
En sandbox, utilisez POST /api/v1/payments/:id/confirm pour simuler manuellement la confirmation d'un paiement en attente.

Idempotence

Passez le header Idempotency-Key: <uuid-unique> dans chaque requête d'initiation. Si la même clé est envoyée deux fois, la deuxième requête retourne le résultat de la première sans créer un doublon.

Remboursements & Litiges

Remboursement

POST /payments/:id/refund — remboursement total ou partiel. L'admin valide depuis le dashboard. Le cashback est réduit proportionnellement.

Litige (Chargeback)

POST /payments/:id/dispute — initie une procédure de contestation. Suivi depuis /admin/disputes.

Système de Fidélité

Le moteur de fidélité (lib/loyalty-engine.js) évalue automatiquement chaque client et met à jour son niveau de cashback.

Évaluation des niveaux

NiveauCashbackCondition (achats)Fenêtre
OPEN0%Moins de 3 achats—
LIVE5%≥ 3 achats ou montant seuil3 derniers mois
GOLD8%≥ 6 achats ou montant seuil6 derniers mois
ROYAL12%≥ 12 achats ou montant seuil12 derniers mois

Batch nocturne

Chaque nuit à 2h00, le worker workers/loyalty-batch.js évalue tous les clients actifs :

  1. Calcule achats et montants sur la fenêtre glissante du tier
  2. Détermine si le client monte, reste stable, ou descend de tier
  3. Met à jour le statut en base
  4. Envoie une notification SMS/Email si changement de tier
  5. Déclenche le webhook loyalty.status_changed si un marchand écoute

Surcharges par pays

L'admin peut configurer des taux Y% différents par pays via PUT /api/v1/loyalty/config-country/:countryId/:status. Par exemple, les clients au Kenya (KES) peuvent avoir un taux ROYAL de 10% au lieu de 12%.

Portefeuille (Wallet)

Chaque client possède un portefeuille virtuel où s'accumulent ses cashbacks. Il peut les consulter sur son dashboard et les utiliser lors de prochains achats (selon configuration marchand).

Anti-Fraude

Le module lib/fraud.js analyse chaque transaction avant traitement et lui attribue un score de risque de 0 à 100.

Règles disponibles

RègleDescription
max_amountMontant maximum autorisé par transaction
max_tx_per_hourNombre max de transactions par heure (par client/téléphone)
max_tx_per_dayNombre max de transactions par jour
max_daily_volumeVolume total max par jour pour un client
blacklist_phoneBlocage immédiat si le numéro est en liste noire

Comportement selon le score

Score 0–30 → Transaction autorisée
Score 31–69 → Transaction autorisée avec log d'alerte
Score >= 70 → Transaction BLOQUÉE
→ Alerte temps réel envoyée à l'admin (SSE + email)
→ Webhook fraud.blocked (si configuré)
→ Log dans la table fraud_alerts

Blacklist téléphones

L'admin peut ajouter/retirer un numéro via POST /api/v1/fraud/blacklist/phone. Tout paiement depuis un numéro blacklisté est immédiatement rejeté (score 100).

Seuils marchand

Chaque marchand peut avoir des limites spécifiques : max_transaction_amount, daily_volume_limit, et activer/désactiver le allow_guest_mode (paiements sans compte client).

!
Les alertes fraude en temps réel nécessitent une connexion SSE active (GET /api/v1/sse). Le dashboard admin affiche ces alertes automatiquement.

Signature et sécurité

Chaque payload webhook est signé avec HMAC-SHA256. Vérifiez toujours la signature avant de traiter un webhook.

const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(payload))
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected)
  );
}

// Dans votre endpoint
app.post('/webhook', (req, res) => {
  const sig = req.headers['x-afrikfid-signature'];
  if (!verifyWebhook(req.body, sig, process.env.WEBHOOK_SECRET)) {
    return res.status(401).send('Invalid signature');
  }
  // Traiter l'événement...
  res.sendStatus(200);
});

Mise à jour automatique des taux

Un worker (workers/exchange-rates.js) s'exécute chaque heure et met à jour les taux depuis :

  1. Open Exchange Rates (si OPENEXCHANGERATES_APP_ID configuré)
  2. Fixer.io (si FIXER_API_KEY configuré)
  3. Taux manuels depuis le dashboard admin Taux de Change

L'admin peut également modifier les taux manuellement depuis /admin/exchange-rates ou via PUT /api/v1/reports/exchange-rates/:currency.

const crypto = require('crypto');

class AfrikFidClient {
  constructor(baseUrl, apiKey) {
    this.baseUrl = baseUrl.replace(/\/$/, '');
    this.apiKey = apiKey;
    this.accessToken = null;
    this.refreshToken = null;
  }

  async _request(method, path, data = null, extraHeaders = {}) {
    const headers = {
      'Content-Type': 'application/json',
      'X-API-Key': this.apiKey,
      ...extraHeaders,
    };
    if (this.accessToken) {
      headers['Authorization'] = `Bearer ${this.accessToken}`;
    }
    const opts = { method, headers };
    if (data) {
      opts.body = JSON.stringify(data);
    }
    const res = await fetch(`${this.baseUrl}${path}`, opts);
    const json = await res.json();
    if (!res.ok) {
      const err = new Error(json.message || `HTTP ${res.status}`);
      err.status = res.status;
      err.body = json;
      throw err;
    }
    return json;
  }

  async login(email, password) {
    const data = await this._request('POST', '/auth/merchant/login', { email, password });
    this.accessToken = data.access_token;
    this.refreshToken = data.refresh_token;
    return data;
  }

  async refreshAccessToken() {
    const data = await this._request('POST', '/auth/refresh', {
      refresh_token: this.refreshToken,
    });
    this.accessToken = data.access_token;
    return data;
  }

  async initiatePayment(data) {
    const idempotencyKey = data.idempotency_key || crypto.randomUUID();
    return this._request('POST', '/payments/initiate', data, {
      'Idempotency-Key': idempotencyKey,
    });
  }

  async getTransactionStatus(id) {
    return this._request('GET', `/payments/${id}/status`);
  }

  async createPaymentLink(data) {
    return this._request('POST', '/payment-links', data);
  }

  static verifyWebhook(payload, signature, secret) {
    const expected = crypto
      .createHmac('sha256', secret)
      .update(typeof payload === 'string' ? payload : JSON.stringify(payload))
      .digest('hex');
    try {
      return crypto.timingSafeEqual(
        Buffer.from(signature, 'hex'),
        Buffer.from(expected, 'hex')
      );
    } catch {
      return false;
    }
  }
}

import hashlib
import hmac
import json
import uuid
import requests


class AfrikFidClient:
    def __init__(self, base_url: str, api_key: str):
        self.base_url = base_url.rstrip('/')
        self.api_key = api_key
        self.access_token = None
        self.refresh_token = None

    def _headers(self, extra: dict = None) -> dict:
        headers = {
            'Content-Type': 'application/json',
            'X-API-Key': self.api_key,
        }
        if self.access_token:
            headers['Authorization'] = f'Bearer {self.access_token}'
        if extra:
            headers.update(extra)
        return headers

    def _request(self, method: str, path: str, data: dict = None, extra_headers: dict = None):
        url = f'{self.base_url}{path}'
        headers = self._headers(extra_headers)
        response = requests.request(method, url, json=data, headers=headers)
        response.raise_for_status()
        return response.json()

    def login(self, email: str, password: str) -> dict:
        result = self._request('POST', '/auth/merchant/login', {
            'email': email,
            'password': password,
        })
        self.access_token = result['access_token']
        self.refresh_token = result['refresh_token']
        return result

    def refresh_access_token(self) -> dict:
        result = self._request('POST', '/auth/refresh', {
            'refresh_token': self.refresh_token,
        })
        self.access_token = result['access_token']
        return result

    def initiate_payment(self, data: dict) -> dict:
        idempotency_key = data.pop('idempotency_key', str(uuid.uuid4()))
        return self._request(
            'POST', '/payments/initiate', data,
            extra_headers={'Idempotency-Key': idempotency_key}
        )

    def get_transaction_status(self, transaction_id: str) -> dict:
        return self._request('GET', f'/payments/{transaction_id}/status')

    def create_payment_link(self, data: dict) -> dict:
        return self._request('POST', '/payment-links', data)

    @staticmethod
    def verify_webhook(payload, signature: str, secret: str) -> bool:
        if isinstance(payload, dict):
            payload = json.dumps(payload, separators=(',', ':'), sort_keys=False)
        if isinstance(payload, str):
            payload = payload.encode('utf-8')
        expected = hmac.new(
            secret.encode('utf-8'),
            payload,
            hashlib.sha256
        ).hexdigest()
        return hmac.compare_digest(expected, signature)

<?php

class AfrikFidClient
{
    private string $baseUrl;
    private string $apiKey;
    private ?string $accessToken = null;
    private ?string $refreshToken = null;

    public function __construct(string $baseUrl, string $apiKey)
    {
        $this->baseUrl = rtrim($baseUrl, '/');
        $this->apiKey = $apiKey;
    }

    private function request(string $method, string $path, array $data = null, array $extraHeaders = []): array
    {
        $url = $this->baseUrl . $path;

        $headers = [
            'Content-Type: application/json',
            'X-API-Key: ' . $this->apiKey,
        ];
        if ($this->accessToken) {
            $headers[] = 'Authorization: Bearer ' . $this->accessToken;
        }
        foreach ($extraHeaders as $key => $value) {
            $headers[] = "$key: $value";
        }

        $ch = curl_init($url);
        curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
        curl_setopt($ch, CURLOPT_HTTPHEADER, $headers);
        curl_setopt($ch, CURLOPT_CUSTOMREQUEST, strtoupper($method));

        if ($data !== null) {
            curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
        }

        $response = curl_exec($ch);
        $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
        curl_close($ch);

        $decoded = json_decode($response, true);
        if ($httpCode >= 400) {
            $message = $decoded['message'] ?? "HTTP $httpCode";
            throw new RuntimeException($message, $httpCode);
        }
        return $decoded;
    }

    public function login(string $email, string $password): array
    {
        $result = $this->request('POST', '/auth/merchant/login', [
            'email'    => $email,
            'password' => $password,
        ]);
        $this->accessToken  = $result['access_token'];
        $this->refreshToken = $result['refresh_token'];
        return $result;
    }

    public function refreshAccessToken(): array
    {
        $result = $this->request('POST', '/auth/refresh', [
            'refresh_token' => $this->refreshToken,
        ]);
        $this->accessToken = $result['access_token'];
        return $result;
    }

    public function initiatePayment(array $data): array
    {
        $idempotencyKey = $data['idempotency_key'] ?? sprintf('%04x%04x-%04x-%04x-%04x-%04x%04x%04x',
            mt_rand(0, 0xffff), mt_rand(0, 0xffff),
            mt_rand(0, 0xffff),
            mt_rand(0, 0x0fff) | 0x4000,
            mt_rand(0, 0x3fff) | 0x8000,
            mt_rand(0, 0xffff), mt_rand(0, 0xffff), mt_rand(0, 0xffff)
        );
        unset($data['idempotency_key']);
        return $this->request('POST', '/payments/initiate', $data, [
            'Idempotency-Key' => $idempotencyKey,
        ]);
    }

    public function getTransactionStatus(string $id): array
    {
        return $this->request('GET', "/payments/$id/status");
    }

    public function createPaymentLink(array $data): array
    {
        return $this->request('POST', '/payment-links', $data);
    }

    public static function verifyWebhook(string $payload, string $signature, string $secret): bool
    {
        $expected = hash_hmac('sha256', $payload, $secret);
        return hash_equals($expected, $signature);
    }
}
